# ADR 0025: Live model registry moves to the user data dir, out of the git work tree

## Status

Accepted 2026-06-11

## Context

The live registry (`registry_path()`) resolved to
`~/local-llm/config/models.yaml` — a git-tracked file inside the work tree.
Every `scan` and every load-status flip mutates it at runtime, so the tree
was permanently dirty, every diff carried registry noise, and runtime state
repeatedly snuck into commits (it already bit us: an MCP `scan_models`
clobbered curated entries — audit fix A2 added merge + `.bak`, but the file
still lived in the tree). The work tree is also disposable in ways state
must not be: `git clean -fdx` is a routine move in a Rust repo and would
have deleted both the registry and its `.bak`; `lamu agent`'s bubblewrap
sandbox mounts the cwd read-write, so sandboxed agents could corrupt it;
fresh worktrees each saw a divergent copy.

The codebase already had two data-dir precedents: `scheduler_lock` uses
`~/.local/state/lamu/` and `media_paths` uses `~/.local/share/lamu/`.

## Decision

`registry_path()` resolves to `~/.local/share/lamu/models.yaml`
(`dirs::data_dir()`), overridable via `$LAMU_REGISTRY`. The tracked file is
renamed to `config/models_default.yaml` and becomes a read-only seed.
`ensure_registry()` runs at CLI entry (best-effort) and seeds the live file
on first run, preferring the legacy in-tree `config/models.yaml` (so a
pre-0025 setup migrates its curated `main`/`speculative`/`sampling`/`notes`
verbatim) over the tracked seed. No source present is fine — the first
`lamu scan` creates the file. The `scan` backup (`.bak`) follows the live
path into the data dir automatically.

## Rationale

- Runtime state does not belong in a source tree. Registry mutations are
  routine (every load flips `status:`); a git-tracked live file guarantees
  permanent `git status` noise and accidental commits of machine state.
- `git clean -fdx` / repo moves / re-clones must not destroy the model
  registry. The data dir survives all three.
- `lamu agent` sandboxes agents to cwd-rw (no `$HOME`); a registry outside
  the cwd is structurally protected from agent mutation rather than
  protected by convention.
- Matches the existing `scheduler_lock` / `media_paths` precedents — the
  registry was the last mutable state living in the tree.
- Migration-preferring-legacy keeps the one existing install's curated
  entries without a manual step.

## Alternatives Considered

- **Gitignore in place** (untrack `config/models.yaml`, commit an example) —
  minimal change, but the live file stays inside the work tree: `git clean
  -fdx` deletes it (and its `.bak` beside it), fresh clones/worktrees have
  no registry until the same seed code is written anyway, and the agent
  sandbox can still write it. Same code, worse location.
- **Status quo + `.bak`** (audit A2 alone) — softens corruption but leaves
  the dirty-tree problem, which was the actual complaint, unsolved forever.
- **`~/.local/state/lamu/`** (like the scheduler lock) — XDG semantics put
  *state that can be regenerated* in state dirs; the registry carries
  curated, hand-edited configuration (main/speculative/sampling/notes) that
  cannot be regenerated by a scan, so `data_dir` is the better fit (and
  matches `media_paths`).

## Consequences

- `config/models_default.yaml` (seed) and the live file can drift: models
  added live never back-propagate. The seed is a bootstrap, not a mirror —
  keep it minimal; `scan` populates real installs.
- Muscle memory and external scripts pointing at
  `~/local-llm/config/models.yaml` break once. The TUI settings label and
  test helpers now point at the new path; `$LAMU_REGISTRY` exists for
  anything that needs a custom location.
- `ensure_registry()` is best-effort at CLI entry: a seeding failure warns
  and continues (an empty registry degrades gracefully; refusing to start
  would break cloud-only use).
- Tests gain a clean isolation seam: point `$LAMU_REGISTRY` at a tempdir.

## Related Decisions

ADR 0009 (confined media output paths — the `data_dir` precedent),
ADR 0014 (~/local-llm root paths — now code+seed only, not live state),
ADR 0023 (module architecture), audit fix A2 (scan merge + `.bak`).

## Validation

- `cargo test -p lamu-core` covers: path resolves outside the work tree,
  `$LAMU_REGISTRY` override wins, seeding prefers legacy over seed, seeding
  is idempotent, and no-source is a clean no-op.
- E2E: run `lamu scan`; `git status` must stay clean and the `.bak` must
  appear next to the live file in `~/.local/share/lamu/`.
- Revisit trigger: a multi-machine sync story for the registry (would need
  an explicit export/import, not a return to the work tree).
